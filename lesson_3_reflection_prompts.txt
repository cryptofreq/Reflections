When would you want to use a remote repository rather than keeping all your work local?
It can serve as an additional storage area or backup.  It comes in handy for geographically
dispersed software development teams.  It provides a local working area for all

Why might you want to always pull changes manually rather than having Git automatically stay up-to-date with your remote repository?
Its a matter of control.  Computers aren't very good at mindreading.  If it was automatic then
automatic updates would take control away from the developer.

Describe the differences between forks, clones, and branches. When would you use one instead of another?
 A fork is the process of cloning somebody's repository on github. A clone is when you make a copy of a fork onto your local pc
 A branch is when you want to set up an alternate development path that coincides side by side with the master branch.
 You would fork somebody's repository to make changes to it without interfering with their work.  You would clone a repository
 when you want to make a copy of a repository on your local pc in order to make changes or add a branch.

What is the benefit of having a copy of the last known state of the remote stored locally?
Its always a good idea to have a copy of the "last known good" version of a file that you can fall back on.
